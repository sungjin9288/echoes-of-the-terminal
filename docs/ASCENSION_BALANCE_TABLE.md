# Ascension Balance Table (v3)

기준일: 2026-03-05  
단일 기준 소스: `progression_system.ASCENSION_TABLE`, `progression_system.get_ascension_profile()`

## 컬럼 설명
- `Penalty+`: 오답 기본 페널티 추가값
- `EasyGlitch`: Easy 노드 글리치 강제 여부
- `TimeDelta`: 기본 제한시간 보정(초)
- `Timeout+`: 타임아웃 페널티 추가값
- `StartTrace`: 런 시작 추적도
- `Shopx`: 중간 상점 가격 배율
- `Rewardx`: 런 정산 보상 배율
- `Bossx`: 보스 오답 페널티 배율
- `BossPhases`: 보스 클리어에 필요한 페이즈 수
- `BossTimeΔ/Phase`: 보스 페이즈가 오를 때마다 추가 적용되는 제한시간 보정(초)
- `BossStep`: 보스 페이즈가 오를 때마다 누적되는 보스 페널티 추가 배율
- `LockCat`: `cat log` 명령 차단이 시작되는 보스 페이즈 (없으면 99)
- `LockSkill`: `skill` 명령 차단이 시작되는 보스 페이즈 (없으면 99)
- `CmdPenalty`: 차단 명령 사용 시 즉시 부과되는 TRACE 패널티
- `FakeKW`: 보스 진입 시 표기되는 가짜 키워드 수
- `RouteElite`: NORMAL이 ELITE로 치환될 확률
- `ReliefDecay`: REST/SHOP이 NORMAL로 치환될 확률
- `MinElite`: 분기 선택지 전체에서 보장되는 최소 ELITE 수

## 티어 규칙
- ASC 0~9: 경제/보스/경로 배율 보정 없음
- ASC 10~14: 상점가 ×1.15, 보상 ×0.95, 보스 페널티 ×1.10
- ASC 12~14: 경로 변이 시작 (ELITE 확률 15%, 휴식/상점 약화 10%, 최소 ELITE 1개 보장)
- ASC 15~19: 상점가 ×1.30, 보상 ×0.90, 보스 페널티 ×1.20, 경로 변이 강화
- ASC 18~19: 보스 2페이즈(페이즈당 제한시간 -2, 보스 페널티 추가 배율 +0.10)
- ASC 20: 상점가 ×1.50, 보상 ×0.85, 보스 페널티 ×1.35, 보스 3페이즈(+0.12), 페이즈 명령 잠금(cat log/skill), 가짜 키워드 경보, 경로 변이 최대치
- ASC 20 보스 페이즈 텍스트/정답은 `boss_phase_pack.json` 오버라이드가 있으면 해당 데이터를 우선 사용

## 운영 가이드
- 실제 제한시간/타임아웃은 퍼크/아티팩트와 합산 적용된다.
- 제한시간은 최소 8초, 타임아웃 페널티는 최소 1로 클램프된다.
- Ascension 해금은 "현재 해금 레벨 이상에서 승리 시 다음 레벨 +1" 규칙을 따른다.
